# First Note

这是一个最小试跑材料。

主题：AI 工作系统为什么需要 personal wiki。

核心观点：

- 对话上下文会过期，文件系统不会。
- 输入材料应该先进入 `wiki/raw/` 和 `wiki/sources/`，再被用于输出。
- `active-context.md` 只保存当前 1-2 周仍有用的上下文。
- `friction-log.md` 记录系统摩擦，用来改流程，而不是评价人。

待验证问题：

- 这个最小结构是否足够让 Codex 完成一次“输入 -> personal wiki”的闭环？
- 是否需要把 pod2wiki 作为默认安装依赖，还是只保留为可选输入模块？
